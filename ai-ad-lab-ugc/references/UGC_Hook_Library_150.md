UGC HOOK PATTERN LIBRARY (155 PATTERNS)
=========================================

⚠️  CRITICAL — READ THIS BEFORE USING ANY PATTERN  ⚠️
═══════════════════════════════════════════════════════

**These patterns are SHAPES, not fill-in-the-blank templates.**

The single biggest mistake when using this library is grabbing a pattern like
`"If you're still dealing with [pain], watch this"` and stuffing VOC keywords
into the bracket. That produces robotic lines like:

❌ "If you're still dealing with inconsistent creative output and wasted ad budget, watch this"

That is NOT a usable hook. It's keyword-stuffing. It sounds written, not spoken.

Instead, read the pattern, understand its SHAPE and its EMOTIONAL MOVE, then write
a line in natural human speech that performs the same move:

✅ "If you're still paying people to make bad ads — stop. Watch this."
✅ "Honestly, if your ads still aren't working, I have something to tell you."
✅ "So if you're the person still burning money on ads that flop — yeah, this is for you."

All three use pattern B1 as a shape. None of them paste raw VOC words into brackets.
All of them sound like something a real person would actually say.

**The rule: never output a hook that still has brackets, reads like ad copy, or
stacks VOC keywords in sequence. Rewrite every pattern into natural speech before
using it in a script. See `Natural_Speech_Rules.md` → "The Hook Pattern Translation Move"
for the full rewrite guide.**

═══════════════════════════════════════════════════════

Every hook you generate MUST use a pattern from this library as its shape. You may NOT
invent new pattern shapes. You MUST rewrite every pattern into natural spoken English
before using it — the raw pattern is a starting structure, never a finished line.

You ARE allowed to generate multiple variations of the same pattern (different emotional
framing, different opening phrases, different rhythms) — and you SHOULD, especially for
the 2 alternative hook lines per script.
